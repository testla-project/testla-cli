# Test Generation with Playwright-CLI (Screenplay Pattern)

This guide explains how to use Playwright-CLI to **discover selectors and actions**, and how to correctly translate them into **Screenplay-based tests** using testla-screenplay-playwright.

> ⚠️ Important  
> The code generated by Playwright-CLI is **never copied directly into a test file**.  
> It is used only to:
> - discover stable selectors
> - understand the correct Playwright locator strategy
> - model Screens and Tasks

---

## Golden Rules

- Generated Playwright code must NEVER appear in `.spec.ts` files.
- Tests must ONLY contain Actors, Tasks, and Questions.
- Selectors discovered via CLI must be stored in Screen classes.
- Recorded interaction sequences must be translated into Tasks.

---

## Mental Model

> Playwright-CLI shows **how** to interact with the page.  
> Screens define **where** elements are.  
> Tasks define **what** happens.  
> Tests define **who** performs the Tasks.

---

## 1. Inspect the UI with Playwright-CLI

Start an interactive session:
```bash
playwright-cli open https://example.com/login
playwright-cli snapshot
```

The snapshot reveals semantic locators, e.g.:
```text
e1 [textbox "Email"]
e2 [textbox "Password"]
e3 [button "Sign In"]
```

Perform actions:
```bash
playwright-cli fill e1 "user@example.com"
playwright-cli fill e2 "password123"
playwright-cli click e3
```

Playwright-CLI outputs:
```typescript
await page.getByRole('textbox', { name: 'Email' }).fill('user@example.com');
await page.getByRole('textbox', { name: 'Password' }).fill('password123');
await page.getByRole('button', { name: 'Sign In' }).click();
```

## 2. Translate Locators into a Screen

From the discovered locators, create Screen selectors.
```typescript
static EMAIL_INPUT: LazySelector = (page) =>
    page.getByLabel('Email');

static PASSWORD_INPUT: LazySelector = (page) =>
    page.getByLabel('Password');

static SIGN_IN_BUTTON: LazySelector = (page) =>
    page.getByRole('button', { name: 'Sign In' });
```

## 3. Translate the Interaction into a Task

The recorded sequence becomes a Task.
```typescript
public async performAs(actor: Actor): Promise<any> {
    return actor.attemptsTo(
        Enter.value(this.username).into(LoginScreen.EMAIL_INPUT),
        Enter.value(this.password).into(LoginScreen.PASSWORD_INPUT),
        Click.on(LoginScreen.SIGN_IN_BUTTON),
    );
}
```

## 4. Write the Screenplay Test

A test never contains Playwright API calls.
```typescript
import { test } from '../fixtures/user';
import { Element } from '@testla/screenplay-playwright/web';
import { Login } from '../screenplay/tasks';
import { DashboardScreen } from '../screenplay/screens';

test('Login as Paul', async ({ Paul_Private }) => {
    await Paul_Private.attemptsTo(
        Login.withCredentials('user@example.com', 'password123')
    );

    await Paul_Private.asks(
        Element.toBe.visible(DashboardScreen.HEADER)
    );
});
```

## What must NEVER happen

❌ Copy page.getByRole(...) into tests
❌ Write selectors directly into Tasks
❌ Use Playwright API inside .spec.ts
❌ Skip Screens and go directly from CLI to test

## Workflow Summary
* Use Playwright-CLI to explore the page.
* Extract semantic locators.
* Create/update Screen classes.
* Convert the recorded flow into a Task.
* Write a Screenplay test using Actors and Tasks.
